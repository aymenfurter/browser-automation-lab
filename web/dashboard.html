LangGraph: Manual Compaction

View on GitHub

LangGraph gives you full control over context management. When messages exceed a threshold, a compaction node summarizes older messages into a progress report and removes the originals using RemoveMessage. This keeps the context window bounded while preserving task state across 20 search iterations.

Docs: StateGraph | Conditional Edges | Managing Conversation History | Tool Calling | Playwright Page API

langgraph_agent/compaction.py 
COMPACTION_THRESHOLD = 15

def should_compact(state: AgentState) -> str:
    """Conditional edge: route to 'compact' or 'continue'."""
    if len(state["messages"]) > COMPACTION_THRESHOLD:
        return "compact"
    return "continue"

async def compact_messages(state: AgentState) -> dict:
    """Summarize older messages and remove them from state."""
    messages = state["messages"]
    keep_recent = 8

    # Find safe cut point - never split tool_calls/tool_response pairs
    candidate_end = len(messages) - keep_recent
    while candidate_end > 1:
        msg = messages[candidate_end]
        if isinstance(msg, ToolMessage):
            candidate_end -= 1
        elif isinstance(msg, AIMessage) and msg.tool_calls:
            candidate_end -= 1
        else:
            break

    old_messages = messages[1:candidate_end]
    summary_response = await llm.ainvoke([
        SystemMessage(content="Summarize completed searches and results...")
    ] + old_messages)

    # Remove old messages, insert summary
    delete_ops = [RemoveMessage(id=m.id) for m in old_messages]
    summary_msg = SystemMessage(
        content=f"[PROGRESS SO FAR]:\n{summary_response.content}"
    )
    return {"messages": delete_ops + [summary_msg]}
langgraph_agent/graph.py - Graph Structure 
graph = StateGraph(AgentState)
graph.add_node("agent", agent_node)      # LLM reasoning
graph.add_node("tools", ToolNode(tools)) # Playwright execution
graph.add_node("compact", compact_messages)

graph.set_entry_point("agent")
graph.add_conditional_edges("agent", should_continue, {
    "tools": "tools",  # Agent wants to call a tool
    "done": END,       # Agent is finished
})
graph.add_conditional_edges("tools", after_tools, {
    "compact": "compact",  # Too many messages -> summarize
    "continue": "agent",   # Under threshold -> continue
})
graph.add_edge("compact", "agent")  # After compaction -> back to agent

GitHub Copilot SDK: Automatic Compaction

View on GitHub

The Copilot SDK handles compaction automatically via infinite sessions. When the context window fills up, the SDK transparently compacts the conversation in the background. You simply receive SessionCompactionStart / SessionCompactionComplete events - no manual summarization logic needed.

Docs: GitHub Copilot | Copilot Extensions | Copilot SDK (Python) | Agent Communication | Playwright Page API

copilot_sdk_agent/agent.py 
from copilot import CopilotClient, define_tool
from copilot.session import PermissionHandler

async with CopilotClient() as client:
    async with await client.create_session(
        model="gpt-4.1",
        on_permission_request=PermissionHandler.approve_all,
        tools=[navigate, click, fill_input, press_key,
               wait_for_selector, extract_links],
    ) as session:
        done = asyncio.Event()

        def on_event(event):
            data = event.data
            if isinstance(data, SessionCompactionStartData):
                print("Compaction started (automatic)")
            elif isinstance(data, SessionCompactionCompleteData):
                print("Compaction complete")
            elif isinstance(data, SessionIdleData):
                done.set()

        session.on(on_event)
        await session.send(task_prompt)
        await done.wait()  # SDK manages context automatically
Tool Definition (Pydantic schema) 
from copilot import define_tool
from pydantic import BaseModel, Field

class NavigateParams(BaseModel):
    url: str = Field(description="URL to navigate to")

@define_tool(description="Navigate the browser to a URL.")
async def navigate_browser(params: NavigateParams) -> str:
    await page.goto(params.url, wait_until="domcontentloaded")
    title = await page.title()
    return f"Navigated to: {title} ({page.url})"

Key Difference: Compaction Strategy

Full README

Both approaches solve the same problem: browser automation generates many messages (tool calls + results + screenshots context). After ~15 messages, the context window pressure becomes significant.

Aspect LangGraph (Manual) Copilot SDK (Automatic)
When it triggers Developer-defined threshold (15 msgs) SDK decides based on token budget
What it preserves Custom summary + last 8 messages SDK-managed (opaque)
Developer control Full (custom summarization prompt) None (fully managed)
Safety constraint Must not split tool_call/response pairs Handled by SDK
Code complexity ~60 lines of compaction logic Zero (built-in)

Infrastructure: Both agents use Azure OpenAI (gpt-4.1) deployed via Bicep. The Copilot SDK authenticates via gh auth (local dev) while LangGraph uses DefaultAzureCredential with RBAC.
